# Building from Source

## Prerequisites

- **Node.js** v18 or later (v20+ recommended)
- **npm** v9 or later
- **Git**
- **macOS, Linux, or Windows** (Electron supports all three; some features like `safeStorage` behave differently per platform)
- **Optional:** `mpv` and `ffmpeg` for media overlay and screen recording features
- **Optional:** `yt-dlp` for stream extraction in the media engine

---

## Clone and Install

```bash
git clone <repo-url> tappi-browser
cd tappi-browser
npm install
```

`npm install` fetches all runtime and dev dependencies. The `electron` package (~100 MB) is downloaded as part of this step.

---

## Build

```bash
npm run build
```

This runs:
```
tsc && cp -r src/ui dist/ && cp src/content-preload.js dist/ && chmod +x dist/cli.js
```

- TypeScript is compiled from `src/` to `dist/` (see `tsconfig.json`).
- The `src/ui/` directory (HTML/CSS/JS for the browser chrome) is copied to `dist/ui/`.
- `src/content-preload.js` (injected into web pages for DOM access) is copied to `dist/`.
- `dist/cli.js` is made executable.

Output directory: `dist/`

---

## Run

```bash
npm start
```

This runs `npm run build && electron dist/main.js`. It rebuilds first, then launches Electron.

For iterative development without rebuilding:
```bash
npm run dev
```

`dev` does the same as `start` — full build then launch. There is no file-watcher mode; you must re-run `npm run dev` after source changes.

## TypeScript Configuration

From `tsconfig.json`:

| Option | Value | Note |
|--------|-------|------|
| `target` | `ES2022` | Modern JS features |
| `module` | `commonjs` | Required for Electron/Node |
| `outDir` | `dist` | Build output |
| `rootDir` | `src` | Source root |
| `strict` | `true` | Full strict mode |
| `esModuleInterop` | `true` | CommonJS interop |
| `sourceMap` | `true` | Source maps for debugging |

## Dev CLI Server (Port 18900)

When Developer Mode is enabled, a TCP server starts on `localhost:18900` for sending commands from a terminal:

```bash
echo "elements" | nc localhost 18900
echo "navigate to https://news.ycombinator.com" | nc localhost 18900
```

This is useful for scripting, debugging, and integration testing without going through the UI.

Port 18900 is only bound when Developer Mode is on. If the port is already in use, the server silently skips.

---

## Data Directory

User data is stored under `~/.tappi-browser/`:

| Path | Contents |
|------|---------|
| `config.json` | Main configuration (LLM provider, Developer Mode, etc.) |
| `tappi.db` | SQLite database (history, bookmarks, conversations) |
| `tools.json` | Registered CLI tools |
| `api-keys.json` | Encrypted API keys (via Electron safeStorage) |
| `api-services.json` | Registered API service definitions |
| `cron-jobs.json` | Scheduled jobs |
| `api-token` | Bearer token for HTTP API (mode 0600) |

Default workspace: `~/tappi-workspace/`
